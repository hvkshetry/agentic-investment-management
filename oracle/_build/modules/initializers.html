Initializers

The initializers module provides functions for initializing and validating various data structures used in the Oracle system.

Tax Lots

service.initializers.initialize_tax_lots(tax_lots: DataFrame) → DataFrame[source]

Initialize and validate tax lots DataFrame.

Parameters:

  • tax_lots – DataFrame with columns:

  • tax_lot_id (-) – Unique identifier for each tax lot

  • identifier (-) – Security identifier

  • quantity (-) – Number of shares/units

  • cost_basis (-) – Total cost basis of the lot

  • date_acquired (- date or) – Purchase date of the lot

Returns:

  • tax_lot_id (str): Unique identifier for each tax lot (generated if not provided)

  • identifier (str): Security identifier (standardized to uppercase)

  • quantity (float): Number of shares/units (validated as non-negative)

  • cost_basis (float): Total cost basis of the lot (validated as non-negative)

  • date (datetime): Acquisition date (standardized column name from date or date_acquired)

Return type:

Validated tax lots DataFrame with standardized columns and types

Raises:

ValueError – If tax lots DataFrame is missing required columns or contains invalid data

Spreads

service.initializers.initialize_spreads(spreads: DataFrame | None, all_identifiers: Set[str], prices: DataFrame) → DataFrame[source]

Initialize spreads DataFrame. If no spreads provided, create default DataFrame with 0.0001 (1bps) spread for all identifiers.

Parameters:

  • spreads (pd.DataFrame | None) – Optional DataFrame with columns:

  • identifier (-) – Security identifier

  • spread (-) – Bid-ask spread as decimal (e.g., 0.0001 for 1bps)

  • all_identifiers (Set[str]) – Set of all identifiers that need spreads

  • prices (pd.DataFrame) – DataFrame with columns:

  • identifier – Security identifier

  • price (-) – Current price used to calculate per_share_cost

Returns:

  • identifier (str): Security identifier

  • spread (float): Bid-ask spread as decimal

  • per_share_cost (float): Pre-calculated transaction cost per share (price * spread)

Return type:

DataFrame with spreads for all identifiers

Notes

  • Default spread of 0.0001 (1bps) is used for any missing identifiers

  • CASH_CUSIP_ID is automatically added with 0 spread if needed

  • All spreads are validated to be non-negative

Raises:

  • ValueError – If:

  • - Spreads DataFrame is provided but missing required columns –

  • - Contains negative spread values –

  • - Contains duplicate identifiers –

Tax Rates

service.initializers.initialize_tax_rates(tax_rates: DataFrame | None = None) → DataFrame[source]

Initialize and validate tax rates DataFrame.

Parameters:

  • tax_rates (Optional[pd.DataFrame]) – DataFrame with columns:

  • gain_type (-) – Type of gain (‘short_term’, ‘long_term’, ‘qualified_dividend’)

  • federal_rate (-) – Federal tax rate as decimal (0 to 1)

  • state_rate (-) – State tax rate as decimal (0 to 1)

  • total_rate (-) – Combined federal and state rate as decimal (0 to 1)

Returns:

  • Short-term gains: 41% (35% federal + 6% state)

  • Long-term gains: 26% (20% federal + 6% state)

  • Qualified dividends: 21% (15% federal + 6% state)

Return type:

Validated DataFrame with standardized tax rates. If no rates provided, uses defaults

Notes

Default rates are based on 75th percentile wealth users: - Federal ordinary income (short term): 35% - Federal long term capital gains: 20% - Federal qualified dividends: 15% - State tax rate: 6% (average for high tax states)

Raises:

  • ValueError – If:

  • - Required columns are missing –

  • - Invalid gain types are present –

  • - Required gain types are missing –

  • - Tax rates are outside [0,1] range –

  • - Total rate doesn't match federal + state –
